# Style Guide

## Color

| Token | Hex | Usage |
| --- | --- | --- |
| Bone | `#F7F3EC` | primary light background |
| Paper | `#EEE6DA` | secondary surface and print support |
| Graphite | `#181C20` | primary dark/base ink |
| Slate | `#313842` | secondary dark surface |
| Signal Sage | `#6F887D` | brand accent, proof emphasis, focus |
| Oxide | `#A86545` | human accent, secondary emphasis |
| Alert Amber | `#B48B3E` | warnings, flags, attention states only |

## Typography

| Role | Face | Usage |
| --- | --- | --- |
| Display | `Newsreader` | hero titles, section titles, key numerals |
| UI / body | `Public Sans` | descriptions, resume copy, supporting text |
| Data / meta | `IBM Plex Mono` | labels, chips, proof levels, utility controls |

## Logo system

- Primary mark: `GRIGORII` wordmark with `support systems` descriptor
- Secondary mark: geometric monogram with one routing divider cut
- Required contexts: header, resume, favicon, app icon, OG/Twitter image
- Minimum rule: monogram must remain legible at 16px and in monochrome

## Component rules

- Prefer dossier cards, evidence rails, status chips, and screenshot frames
- Keep CTAs mono-cased and operational, not marketing-like
- Use borders and spacing for hierarchy before using color blocks
- Keep the portrait secondary to proof modules

## Motion

- Use short measured reveals and state transitions
- Avoid ambient blobs, endless drift, and decorative glow loops
- Respect reduced motion and keep page transitions functional

## Do / don't

- Do make support evidence the fastest thing to understand
- Do let the builder layer appear as reinforcement, not as a costume
- Do keep the light theme as the canonical identity
- Don’t reintroduce blurred glass surfaces or luxe coral-teal ambience
- Don’t let graphics overpower the hiring signal
